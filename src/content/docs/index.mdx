Unitflow is an Effect-first state manager for UI applications. It keeps Effect's
dependency injection, streams, fibers, scopes, and typed errors in the place
where UI logic usually becomes callbacks and hook glue.

The core idea: describe behavior in small models, expose typed inputs and
outputs, then let React bind the `ui` surface without owning the logic.

## One Flow, Two Models

A project setup flow has two kinds of complexity: one step owns focused form
logic, while the parent owns orchestration. Unitflow keeps those boundaries
visible.

`ProjectNameModel` validates and emits a submitted draft. `ProjectSetupModel`
gets that child model through Effect DI, listens to the child's output as a
stream, combines it with its own template store, and emits a final event.

```ts
import { Effect } from "effect";
import { Event, Model, Store } from "@unitflow/core";

interface ProjectDraft {
  readonly name: string;
}

const normalize = (name: string) => name.trim();
const isProjectNameReady = (name: string) => normalize(name).length >= 3;

export class ProjectNameModel extends Model.Service<ProjectNameModel>()(
  "examples/project-name",
)({
  make: Effect.gen(function* () {
    const name = Store.make("");
    const canSubmit = name.pipe(Store.map(isProjectNameReady));

    const view = Store.combine([name, canSubmit], (name, canSubmit) => ({
      name,
      canSubmit,
    }));

    const setName = Event.setter(name);
    const submitted = yield* Event.make<ProjectDraft>();

    const submit = yield* Event.make<void>().pipe(
      Event.handler(() =>
        Effect.gen(function* () {
          const current = normalize(yield* Store.get(name));
          if (!isProjectNameReady(current)) return;

          yield* Event.emit(submitted, { name: current });
        }),
      ),
    );

    return {
      inputs: { setName, submit },
      outputs: { submitted },
      ui: { view, setName, submit },
    };
  }),
}) {}
```

```ts
import { Effect, Stream } from "effect";
import { Event, Model, Registry, Store } from "@unitflow/core";
import { ProjectNameModel } from "./project-name-model";

export type Template = "blank" | "ifc" | "floorPlan";

interface ProjectCreated {
  readonly name: string;
  readonly template: Template;
}

export class ProjectSetupModel extends Model.Service<ProjectSetupModel>()(
  "examples/project-setup",
)({
  make: Effect.gen(function* () {
    const nameStep = yield* Model.get(ProjectNameModel);

    const template = Store.make<Template>("ifc");
    const status = Store.make<"editing" | "ready">("editing");
    const created = yield* Event.make<ProjectCreated>();

    const view = Store.combine([template, status], (template, status) => ({
      template,
      status,
    }));

    const setTemplate = Event.setter(template);

    yield* Registry.run(
      Event.stream(nameStep.outputs.submitted).pipe(
        Stream.mapEffect(({ name }) =>
          Effect.gen(function* () {
            const currentTemplate = yield* Store.get(template);

            yield* Store.set(status, "ready");
            yield* Event.emit(created, {
              name,
              template: currentTemplate,
            });
          }),
        ),
      ),
    );

    return {
      inputs: { setTemplate },
      outputs: { created },
      ui: { view, setTemplate, nameStep },
    };
  }),
}) {}
```

```tsx
import { View } from "@unitflow/react";
import { ProjectNameModel } from "./project-name-model";
import { ProjectSetupModel, type Template } from "./project-setup-model";

export const ProjectNameStep = View.make(
  ProjectNameModel,
  ({ view, setName, submit }) => (
    <form
      onSubmit={(event) => {
        event.preventDefault();
        submit();
      }}
    >
      <input
        value={view.name}
        onChange={(event) => setName(event.currentTarget.value)}
        placeholder="Project name"
      />

      <button disabled={!view.canSubmit}>Continue</button>
    </form>
  ),
);

export const ProjectSetupView = View.make(
  ProjectSetupModel,
  ({ view, setTemplate, nameStep }) => (
    <section>
      <ProjectNameStep unit={nameStep} />

      <TemplateTabs
        value={view.template}
        onChange={(template: Template) => setTemplate(template)}
      />

      {view.status === "ready" ? <ReadyBanner /> : null}
    </section>
  ),
);
```

```ts
import { assert, it } from "@effect/vitest";
import { Effect, Fiber } from "effect";
import { Event, Model, Registry } from "@unitflow/core";
import { ProjectSetupModel } from "./project-setup-model";

it.effect("creates a project when the child step submits", () =>
  Effect.gen(function* () {
    const setup = yield* Model.get(ProjectSetupModel);
    const created = yield* Effect.fork(Event.waitFor(setup.outputs.created));

    yield* Registry.allSettled(
      Effect.gen(function* () {
        yield* Event.emit(setup.inputs.setTemplate, "floorPlan");
        yield* Event.emit(setup.ui.nameStep.ui.setName, "Lobby refresh");
        yield* Event.emit(setup.ui.nameStep.ui.submit);
      }),
    );

    assert.deepStrictEqual(yield* Fiber.join(created), {
      name: "Lobby refresh",
      template: "floorPlan",
    });
  }).pipe(Effect.provide(testLayer)),
);
```

The UI never subscribes to private state and never imports child internals. It
only receives ports. The parent model owns the flow; the child model owns its
small domain; tests drive public events and assert public outputs.

## Why It Exists

Most UI state tools become a second runtime next to Effect. Unitflow makes the
UI runtime an Effect runtime:

- stores are scoped state, not global singletons
- events are streamable facts and commands, not callback-only wires
- models are Effect-powered controllers with declared ports
- resources and mutations are async recipes over `Effect`, `Store`, and `Event`
- tests run in isolated registries and wait for cascades deterministically

## Reading Order

Start with the primitive data flow:

```txt
Store  -> scoped state
Event  -> streamable discrete messages
Model  -> composition boundary over stores and events
Resource / Mutation -> async reads and writes
```

Then apply it to UI through React Views that only bind declared ports.

## Package Plan

```txt
@unitflow/core    Model, Store, Event, Registry, Resource, Mutation
@unitflow/react   View.make, provider, hooks, and React re-exports
```
